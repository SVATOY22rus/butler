# Дворецкий

"Дворецкий" — это внутренний веб-инструмент для управления доступом к сервисам Linux-сервера по IP-адресам. Проект написан на Flask и SQLite как первый понятный MVP, а позже будет дополнен интеграцией с nftables для реального применения правил доступа.[web:82][web:97][web:107]

## Что уже есть

Сейчас в проекте есть базовый каркас Flask-приложения, структура каталогов, SQLite-база, демо-данные и страницы для сервисов, попыток подключения, белого списка, чёрного списка и журнала действий.[web:82][web:98][web:103]

## Как запустить

### 1. Создать виртуальное окружение

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Инициализировать базу

```bash
flask --app app init-db
```

Команда создаст SQLite-базу и заполнит её тестовыми данными через CLI-команду Flask.[web:82]

### 4. Запустить приложение

```bash
python run.py
```

После этого приложение обычно будет доступно по адресу `http://127.0.0.1:5000`.[web:98]

## Объяснение файлов простыми словами

### `run.py`

Это точка входа. Когда запускается `python run.py`, Python создаёт Flask-приложение и поднимает локальный веб-сервер.

### `app/__init__.py`

Здесь находится функция `create_app()`. Она создаёт приложение, задаёт настройки, подключает базу данных и регистрирует маршруты. Такой способ организации рекомендуется в официальной документации Flask как application factory.[web:97][web:98]

### `app/db.py`

Этот файл отвечает за SQLite:
- открывает соединение с базой;
- создаёт таблицы;
- добавляет демо-данные;
- закрывает соединение после завершения запроса.

Flask рекомендует работать с SQLite через контекст приложения и хранить соединение в `g`, чтобы не открывать базу заново много раз в одном запросе.[web:107]

### `app/routes.py`

Здесь лежат URL-адреса страниц и обработка форм. Например, когда открывается `/services`, Flask запускает функцию `services()` и возвращает HTML-страницу.

### `templates/`

Здесь лежат HTML-шаблоны Jinja2. Базовый шаблон `base.html` задаёт общий каркас страницы, а остальные страницы его расширяют. Такой подход упрощает поддержку интерфейса.[web:102][web:105]
